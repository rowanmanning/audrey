This page is used to test styles for entries, considering many possible ways that websites might mark up common elements (including a few odd ones).

Paragraphs and Inline Text

Links to internal pages look like this: visit the home page. Links to external pages look like this: visit example.com.

Abbreviated text looks like this: RSS.

Strong text looks like this: strong element and b element. Emphasised text looks like this: em element and i element. Combined they look like this: strong + em element.

Line breaks
look like
this.

Citations look like this: RFC 4287. When citations contain a link they look like this: RFC 4287.

Inline code elements look like this: <element>Hi</element>. Split across lines they look like this: <element>This is some long text designed to make the containing element split across lines to demonstrate resilent styling</element>.

Data elements looks like this: thirteen point seven.

Deleted and inserted text looks like this: I hate The Princess Switch I love The Princess Switch.

Keyboard input look like this: press Command + Shift + R to perform a hard refresh in Chrome on Mac OS.

Marked text looks like this: pay attention to this text.

Inline quoted text looks like this: According to Mozilla's website, Firefox 1.0 was released in 2004 and became a big success.

Strikethrough text looks like this: this is no longer relevant.

Sampled output looks like this: you should see something like audited 463 packages in 2.514s, which means that npm has finished installing and auditing your libraries.

Smallprint text looks like this: Audrey is licensed under GPLv3.

Subscript and superscript text looks like this:
subtextsub element and supertextsup element.

Date and time elements look like this: 26th January 1988 and 13:07.

Unarticulated Annotation elements look like this: I make speling mistakes often.

Variable elements look like this: The volume of a box is l × w × h, where l represents the length, w the width and h the height of the box.
